# Workshop 1: Mastering Git Branching and Merging
==========================

Create, merge, and manage branches like a pro with this comprehensive workshop project, focusing on the fundamentals of Git workflow in C.

## Description
This workshop is designed to guide users through the process of creating and managing Git branches, focusing on the critical aspects of version control in software development. It's an essential skill for any developer looking to collaborate efficiently or manage different versions of their codebase.

## Features
- **Branch Creation**: Learn how to create new branches in your Git repository.
- **Merge Operations**: Understand the process of merging changes from one branch into another.
- **Conflict Resolution**: Discover how to resolve conflicts that arise during the merge process.
- **Project Organization**: Improve your repository organization skills by managing different branches for various features or fixes.

## Installation and Usage
1. Clone the repository: `git clone https://github.com/your-username/workshop_1.git`
2. Switch into the project directory: `cd workshop_1`
3. Create a new branch for your feature: `git branch feature/new-feature`
4. Switch to the new branch: `git checkout feature/new-feature`
5. Make changes, commit them, and then merge back into the main branch.

## Project Architecture
The project is structured around the concept of branching and merging. It includes:
- **main**: The primary branch where the final version of the project should reside.
- **feature/new-feature**: An example branch where new features can be developed.
- **fix/bug-fix**: A branch for fixing bugs found in the main branch.
